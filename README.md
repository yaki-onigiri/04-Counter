## アプリ概要

    シンプルなカウンターアプリです。
    ボタンをクリックすることで数値を増減・リセットできます。

    単純なクリック操作に加え、長押しによる連続カウント機能と、長押し継続時間に応じて速度が変化する段階加速処理を実装し、ユーザーの操作効率と操作感を向上させています。

    さらに、カード型レイアウト・ボタン押下アニメーション・数値変化アニメーションなどのUI/UX改善を加え、視認性と操作フィードバックも強化しています。

## Features（主な機能）

---

### 基本機能

    ・カウントの増減（+1 / -1）

    ・「リセット」ボタンでカウントを0に戻す

    ・カウントは0未満にならない（マイナス制限）

### 拡張機能

    ・増減ボタン（+1/-1）両方に長押しによる連続カウント機能を実装

    ・押下開始からの継続時間を計測し、カウント速度が「低速 → 中速 → 高速」と段階的に変化する加速処理を実装

### UX改善（操作性）

    ・マウスをボタン外に移動しても、押している間はカウント継続

    ・マウスを離したタイミングで確実に停止（documentで制御）

    ・長押し時にクリックイベントが重複しないように制御

    ・どのボタンから操作を開始しても、タイマー処理が1つだけ動作するように制御

### UI改善（視認性・操作フィードバック）

    ・カウンター全体をカード型レイアウトで中央配置

    ・背景色とシャドウを追加し、アプリ領域を視覚的に分離

    ・増減/リセットボタンのサイズ・角丸・配色を統一

    ・ボタン押下時に縮小アニメーションを追加

    ・数値変更時に拡大・縮小するアニメーションを追加

    ・数値増加時は緑色、減少時は赤色に一時変化する視覚フィードバックを追加

    ・setTimeout を利用し、一定時間後に色を元に戻すUI演出を実装

### 開発中機能

    ・LocalStorage を使ったカウント値の保存機能

## 設計・実装のポイント

### 課題

    クリック操作のみでは、連続入力時の操作効率が低い

### 解決方法

    長押し状態をフラグで管理し、setTimeout による"再帰的な繰り返し処理"を実装

### 工夫した点

① イベント競合防止・操作安定化

    ・click と mousedown の競合を防ぐため、長押し判定フラグ（isLongPress）を導入

    ・どのボタンから操作を開始してもタイマーが重複しないように、mousedown 時に必ず既存タイマーを初期化

    ・mouseup を document に設定し、ボタン外でマウスを離した場合でも確実に停止できるように制御

② タイマー制御と長押し連続処理

    ・setInterval の固定間隔処理ではなく、setTimeout の再帰処理を採用し、長押し継続時間に応じて待機時間を再計算できる構造に変更

    ・開始・再帰・停止で同一の timerId を管理し、タイマーの多重起動や暴走を防止

    ・Date.now による長押し経過時間を利用し、処理ごとに待機時間を変更することで段階加速を実現

③ コード構造の整理・保守性向上

    ・増算処理と減算処理に分散していた count変更ロジック を changeCount(step) に共通化し、重複コードを削減

    ・単押し処理と長押し連続処理の両方を同じ増減関数で管理する構造に整理し、挙動の統一性と保守性を向上

    ・加速段階の条件判定に使用する"時間条件"と"実行間隔"を定数化し、速度調整や今後の拡張をしやすい構造に改善

④ UIアニメーションとDOMクラス制御

    ・数値変更時に animateCount() を呼び出し、JavaScriptから classList操作 でCSSアニメーションを発火

    ・連続カウント時でも毎回アニメーションが安定して再実行されるように、classのremove → 再描画 → add の流れで制御

    ・button:active を利用し、押下時に縮小する視覚フィードバックを追加し、操作時の反応を判りやすく改善

    ・背景色・カード型レイアウト・ボタン配色を調整し、視認性とアプリ領域の認識しやすさを向上

    ・増加/減少状態を step 引数として関数間で受渡しし、アニメーション色を条件分岐で切り替える構造を実装

    ・setTimeout を利用し、一定時間後に flash用class を削除することで、一時的なUI状態変化を制御

## 学習ポイント

    ・getElementById を使ったDOM取得

    ・addEventListener を使った複数イベント（click / mousedown / mouseup）制御

    ・isPressing / isLongPress を用いた状態管理によるイベント競合防止

    ・setTimeout の再帰呼び出しを利用した長押し連続カウント処理

    ・Date.now を使った押下継続時間の計測と段階加速制御

    ・timerId の一元管理によるタイマー多重起動防止

    ・changeCount(step) による増減ロジックの共通化と保守性向上

    ・classList.add / remove を使った CSSクラスの動的付け/外し

    ・JavaScriptからCSSアニメーションを発火させるDOM制御

    ・@keyframes を用いた数値変化アニメーションの実装

    ・button:active を使った押下時のマイクロインタラクション

    ・連続実行時でもアニメーションを安定再生させる、class再付与の考え方

    ・関数引数（step / type）を利用した状態情報の受け渡し

    ・if文による UI アニメーションの条件分岐制御

    ・setTimeout を利用した一時的なCSS class制御

    ・classList.remove → 再描画 → classList.add によるアニメーション再実行
