## アプリ概要

    シンプルなカウンターアプリです。
    ボタンをクリックすることで数値を増減・リセットできます。

    単純なクリック操作に加え、長押しによる連続カウント機能を実装し、ユーザーの操作効率を向上させています。

## How to Run

1. index.html をブラウザで開く
2. ボタンをクリックしてカウントを操作する

## Features（主な機能）

---

### 基本機能

    ・カウントの増減（+1 / -1）

    ・「リセット」ボタンでカウントを0に戻す

    ・カウントは0未満にならない（マイナス制限）

### 拡張機能

    ・増減ボタン（+1/-1）両方に長押しによる連続カウント機能を実装

    ・長押し継続時間を計測し、カウント速度を段階的に変化させる加速処理を実装

### UX改善

    ・マウスをボタン外に移動しても、押している間はカウント継続

    ・マウスを離したタイミングで確実に停止（documentで制御）

    ・長押し時にクリックイベントが重複しないように制御

    ・どのボタンから操作を開始しても、タイマー処理が1つだけ動作するように制御

### 開発中機能

    ・LocalStorage を使ったカウント値の保存機能

## 設計・実装のポイント

### 課題

    クリック操作のみでは、連続入力時の操作効率が低い

### 解決方法

    長押し状態をフラグで管理し、setTimeout による"再帰的な繰り返し処理"を実装

### 工夫した点

① イベント競合防止・操作安定化

    ・click と mousedown の競合を防ぐため、長押し判定フラグ（isLongPress）を導入

    ・どのボタンから操作を開始してもタイマーが重複しないように、mousedown 時に必ず既存タイマーを初期化

    ・mouseup を document に設定し、ボタン外でマウスを離した場合でも確実に停止できるように制御

② タイマー制御と長押し連続処理

    ・setInterval の固定間隔処理ではなく、setTimeout の再帰処理を採用し、長押し継続時間に応じて待機時間を再計算できる構造に変更

    ・開始・再帰・停止で同一の timerId を管理し、タイマーの多重起動や暴走を防止

    ・Date.now による長押し経過時間を利用し、処理ごとに待機時間を変更することで段階加速を実現

③ コード構造の整理・保守性向上

    ・増算処理と減算処理に分散していた count変更ロジック を changeCount(step) に共通化し、重複コードを削減

    ・単押し処理と長押し連続処理の両方を同じ増減関数で管理する構造に整理し、挙動の統一性と保守性を向上

## 学習ポイント

    ・getElementById を使ったDOM取得

    ・addEventListener によるクリックイベント処理

    ・状態管理（isPressing / isLongPress）による挙動制御

    ・setInterval / clearInterval を使った繰り返し処理

    ・複数イベント（click / mousedown / mouseup）の競合制御

    ・時間（Date.now）を使った状態制御の基礎

    ・setInterval と setTimeout の違いと使い分け

    ・イベント駆動と状態管理を組み合わせたUI制御

    ・setTimeout を用いた「再帰処理による非同期ループ」の実装

    ・タイマーIDの一元管理と停止制御の重要性

    ・Date.now を使った経過時間計測と条件分岐による速度制御

    ・setTimeout の再帰呼び出しによる可変インターバル処理
