### カウンターアプリの基本機能を実装（+1 / -1 / リセット）

▼ 目的
    ボタン操作によって数値を増減・リセットできるアプリを作成する。

---

▼ 仕組み

    カウンターは以下の流れで動く：
        1. 数値（count）を保持する
        2. ボタン操作で数値を変更する
        3. 変更した数値を画面に反映する

    ☞ 「データ変更」と「画面更新」は別処理

---

▼ 具体的な方法

    1⃣ 初期値を用意する
        let count = 0;

    2⃣ HTML要素を取得する
        const countEl = document.getElementById("count");

    3⃣ ボタン操作を検知する
        addEventListener("click", function() {...処理...});

    4⃣ 数値を変更する
        ・count++;
        ・count--;
        ・count = 0;
    
    5⃣ 画面を更新する
        countEl.textContent = count;

---

▼ 使用したコード・技術

    ・getElementById
        ⇒ HTML要素を取得する

    ・addEventListener
        ⇒クリックイベントを検知する

    ・textContent
        ⇒画面のテキストを書き換える
    
    ・関数（updateDisplay）

        function updateDisplay() {
            countEl.textContent = count;
        }

        ⇒表示更新を共通化

---

▼ 重要なポイント

    ・「count（データ）」と「画面表示」は別物
    ・値を変えただけでは画面は変わらない
    ・必ず「表示更新処理」が必要

    値変更 → updateDisplay()

    ☞このセットで動く

---

▼ 学んだこと

    ・JavaScriptは「状態(データ)」を操作してUIに反映する仕組み
    ・イベント(クリック)をきっかけに処理が実行される
    ・同じ処理は関数にまとめることで管理しやすくなる

    ☞基本構造：「イベント → データ変更 → 表示更新」

===

### カウントが0の状態で「-1」ボタンを押してもカウントが反映されない機能を実装

▼ 目的

    カウントが0未満にならないように制御する（マイナス制限の実装）

---

▼ 仕組み

    if文 を使うことで、「特定の条件を満たした場合の未処理を実行する」ことができる
    
    今回は「count が0より大きい場合のみ減算する」という条件で制御する

---

▼ 具体的な方法・使用したコード・技術

    1⃣ カウントを減らす処理（count--）の位置を確認

    2⃣ “数字を減らす処理”の直前に『条件分岐』を入れる
        if (count > 0)

    3⃣ 『条件分岐』の中に“数字を減らす処理”を入れる
        if (count > 0) {
            count--;
        }

    これによって条件を満たしたときだけ実行されるようにする

---

▼ 重要なポイント

    ・「count--;」を“消す”のではなく、if の中に“入れる”
    ・「減らした後に判定」ではなく「減らす前に判定する」ことが重要
    ・『制御（条件分岐）』は「処理の前」に書くこと

---

▼ 学んだこと

    ・シンプルなif文でも、使い方次第で挙動が大きく変わる
    ・バグを防ぐためには「状態（変数）」を直接制御する必要がある
    ・「処理」は“削除する”のではなく、“条件で制御する”ことで安全に実装できる

## 26/04/18

### 長押し機能の安定化（+1 / -1 両方対応・バグ防止）

▼ 目的

    ボタンを長押ししたとき、カウントが連続で増減する機能を実装しつつ、どんな操作（長押し・連打・ボタン切替）でもバグが起きないようにする。

---

▼ 仕組み

    長押しは「ボタンを押している間、同じ処理を繰り返す動き」なので、setInterval を使って一定時間ごとに処理を実行する

    ただし、そのまま使うと以下の問題が起きる
        ・複数の setInterval が同時に動く（暴走）
        ・click と mousedown が動いて二重カウント

    この問題を防ぐために、以下の2つで制御する
        ・isPressing → 今押しているかどうか
        ・isLongPress → 長押しかどうか

▼ 具体的な方法

    1.ボタンを押したとき（mousedown）
    
        ・すでに押していたら何もしない（多重起動防止）

        ・長押しフラグをリセット

        ・setInterval を開始
    
    2.長押し中（setInterval）

        ・長押し状態を true にする

        ・カウント処理を繰り返す
    
    3.ボタンを離したとき（mouseup）

        ・押している状態を解除

        ・setInterval を停止
    
    4.clickイベント

        ・長押し中なら処理を止める（return）

---

▼ 使用したコード・技術

    ⓵
    let isPressing = false;
    let isLongPress = false;
    let intervalId;

    ⓶
    incrementBtn.addEventListener("mousedown", () => {

        clearInterval(intervalId);　// 前処理を止める

        if (isPressing) return;

        isPressing = true;
        isLongPress = false;

        intervalId = setInterval(() => {

            isLongPress = true;

            if (isPressing) {
                count++;
                updateDisplay();
            }
        }, 200);
    });

    ⓷
    document.addEventListener("mouseup", () => {
        isPreesing = false;
        isLongPress = false;

        clearInterval(intervalId);
    });

---

▼ 重要なポイント

    ・setInterval は複数同時に動くため、必ず clearInterval で止める必要がある
    
    ・intervalId は「タイマーを止めるための番号」

    ・mousedown の最初で clearInterval を実行すると、どんな操作でも安定する

    ・click は長押し中でも発火するため、isLongPress で制御しないとバグになる

    ・停止処理は document にまとめることで「取りこぼし」を防げる

---

▼ 学んだこと

    ・「動くコード」と「壊れないコード」は違う

    ・イベント処理は「開始・継続・停止」をセットで考える必要がある

    ・状態（フラグ）で制御すると、複雑な動きでも整理できる

    ・同じ役割の処理（例：interval）は1つにまとめるとバグが減る

    ・エラーやバグの原因は「処理が複数動いている」ケースが多い

===

### 長押し時間を取得し、カウント速度を変える準備をする

▼ 目的

    ボタンの長押し時間に応じて、カウント速度を段階的に変えられるようにするため。そのために「どれくらい押しているか（時間）」を取得する

---

▼ 仕組み

    ボタンを押した瞬間の時間と、話したときの時間を記録し、その差を取ることで「長押し時間」を求める。
    
        長押し時間 = 現在時刻 - 押し始めた時刻
    
    この時間を使って「遅い → 速い」にカウント速度を変える。

---

▼ 具体的な方法

    ① mousedown（押した瞬間）で時間を記録する

    ② mouseup（離した瞬間）で時間の差を計算する

    ③ console.log で値を確認する

---

▼ 使用したコード・技術

    ・押し始めの時間を保存する変数

        let pressStartTime = 0;

    ・ボタンを押したとき（+1/-1 両方）

        incrementBtn.addEventListener("mousedown", () => {
            pressStartTime = Date.now();
        });

        decrementBtn.addEventListener("mousedown", () => {
            pressStartTime = Date.now();
        });
    
    ・ボタンを離したとき

        document.addEventListener("mouseup", () => {
            const duration = Date.now() - pressStartTime;
            console.log(duration);
        });

---

▼ 重要なポイント

    ・Date.now() は「現在の時間（ミリ秒）」を取得する

    ・mousedown と mouseup をセットで使うことで「押していた時間」がわかる

    ・時間はミリ秒なので「1000 = 1秒」

    ・mouseup は document に付けることで、ボタン外でも確実に検知できる

    ・「時間を記録するコード」は mousedown の一番最初に書く。理由としては後に書くと時間にほんの少しのズレが生じるため

---

▼ 学んだこと

    ・『時間を使った制御』は「開始時間を保存して差を取る」ことで実現できる

    ・イベント（mousedown / mouseup）を組み合わせることで状態を管理できる

    ・長押し機能は「時間」と「繰り返し処理」の組み合わせで作られる

    ・今までは「setInterval で固定速度」だったのが、今後は「setTimeout で毎回速度を変える」ことができる

### ② setTimeout を使った連続処理（再帰処理）

▼ 目的

---

▼ 仕組み

    ・setTimeout の中で同じ関数をもう一度呼び出す

---

    1. "カウント処理"を「関数」にする

    function startCountingIncrement() {
        const duration = Date.now() - pressStartTime;
        const interval = getInterval(duration);

        count++;
        updateDisplay();

        timerId = setTimeout(startCountingIncrement, interval);
    }

---

▼ 重要なポイント

    ・white や for ではなく、時間を使ったループ

    ・UI更新やイベント処理（mouseup）が止まらない

---

▼ 学んだこと

    ・setTimeout を使うことで「止められるループ」を作れる。

### ③：setTimeout の再帰処理で毎回速度を作り直す方法

▼ 目的

    長押ししている途中でも、押している時間に応じてカウント速度を変えられるようにするため。

---

▼ 仕組み

    `setInterval`は最初に決めた時間で固定されるため、途中で自然に速度を変えるのが難しい。

    一方、`setTimeout`を使って
    
        「1回処理する → 今の長押し時間を確認する → 次の待機時間を決める → もう1回呼ぶ」
    
    という流れを繰り返すと、毎回違う速度で処理できる。

---

▼ 具体的な方法

    連続カウント関数の最後に`timerId = setTimeout(同じ関数, interval);`を書き、処理が終わるたびに次の実行を予約する形に変更した。

    その都度`interval`を再計算することで、可変速度の連続カウントを実装した。

---

▼ 使用したコード・技術

    ・`setTimeout`

    ・再帰呼び出し

    ・`timerId`によるタイマーID管理

    ・`clearTimeout`による停止

---

▼ 重要なポイント

    固定速度で繰り返す`setInterval`と違い、`setTImeout`の再帰は「毎回待機時間を変えられる」点が大きな違い。

    可変速度を作る場合はこちらの考え方が必要になる。

---

▼ 学んだこと

    同じ繰り返し処理でも、使うタイマー関数によって実装できる内容が変わることを学んだ。

    特に「毎回条件を見ながら次の処理を決める」場合は、setTimeout の再帰が有効だ、と理解した。

### 長押し処理の増算・減算コードを1つの関数に共通化する方法

▼ 目的

    長押し処理の中で、+1用と-1用に似たコードが別々に存在しており、修正箇所が増えていた。
    同じような処理を1つにまとめて、今後の修正や追加機能をしやすくするために共通関数化を行なった。

---

▼ 仕組み

    増算処理と減算処理は別のように見えるが、
    実際には「countの値を変えて画面を更新する」という流れは共通している。

    違うのは「何を足すか」だけなので、
    関数の中身を2つ作るのではなく、外から数字を渡して動作を変える方法を使う。

    この外から渡す数字が step であり、
    ・1 を渡せば1増える
    ・-1 を渡せば1減る
    という仕組みになる。

---

▼ 具体的な方法

    まず count++ や count-- を直接書くのをやめ、
    count変更専用の関数（changeCount(step)）を作成した。

    この関数の中で、
    1. count += step で数値を増減
    2. 0未満になったら0に戻す
    3. 最後に画面表示を更新
    という一連の流れをまとめた。

    その後、
    ・単押しクリック処理
    ・長押し連続処理
    の両方から changeCount() を呼ぶように変更し、
    どちらの操作でも同じ増減ルールが使われる構造にした。

---

▼ 使用したコード・技術

    1.
        function changeCount(step) {
            count += step;

            if(count < 0){
                count = 0;
            }

            updateDisplay();
        }

    2.
        changeCount(1);   // +1
        changeCount(-1);  // -1

    使用技術：
        ・関数の共通化
        ・引数（step）による動作切り替え
        ・count の一元管理

---

▼ 重要なポイント

    changeCount は増算専用・減算専用の関数ではなく、
    「渡された数字の分だけ count を動かす関数」として考える。

    つまり、
    ・1 を渡せば count + 1
    ・-1 を渡せば count - 1
    になる。

    増減の違いは関数の中ではなく、
    外から渡す値で決まるという考え方が重要。

---

▼ 学んだこと

    同じようなコードが複数ある場合、
    違う部分だけを引数として外から渡すと1つの関数にまとめられることを学んだ。

    今後は「増算用」「減算用」とすぐに関数を増やすのではなく、
    処理の共通部分がないかを先に考える意識が必要だと感じた。

### ① 長押し時間による3段階速度変化

▼ 目的

    特定のボタンを押下する時間に応じて、カウント速度が段階的に上がる仕組みを理解する

---

▼ 仕組み

    1. ボタン押下開始時刻を保存する

    2. 連続カウント処理のたびに現在時刻との差から経過時間を出す

    3. 経過時間を getInterval() に渡して待機時間を決める

    4. 待機時間が短くなることでカウント速度が段階的に上がる

---

▼ 具体的な方法

    ・pressStartTime = Date.now(); を押下開始時に記録する位置へ移動した。

    ・startCounting() 内で毎回 duration を再計算し、その値を getInterval(duration) に渡した。

    ・getInterval() で長押し時間に応じて待機時間を3段階に分岐させた。

    ・待機時間を短くすることで、連続カウントの速度が徐々に速くなるようにした。

---

▼ 使用したコード・技術

    ・Date.now()

    ・setTimeout() の再帰処理

    ・getInterval(duration)

    ・if文による条件分岐

    ・長押し継続時間の計測

---

▼ 重要なポイント

    ・押した瞬間の時刻を保存しておかないと正しい経過時間が出せない。

    ・連続処理のたびに経過時間を再計算しないと段階加速にならない。

    ・getInterval() は経過時間を受け取り、待機時間を返す判定用関数。

    ・待機時間が短くなるほどカウント速度は速くなる。

---

▼ 学んだこと

    ・長押し加速は、長押ししている経過時間を毎回確認しながら待機時間を変えることで実現できると理解した。

    ・「時間条件」と「待機時間」の2つを別で考える必要がある。

### ② 加速処理で使用する数値を定数化して管理しやすくする方法

▼ 目的

    加速処理で使用する数値を解りやすく整理し、後から速度調整や機能追加をしやすくする考え方を理解する。

---

▼ 仕組み

    1. 1000 や 250 などの数字をコードに直接書かない。

    2. あらかじめ意味のある名前を付けて定数として保存する。

    3. getInterval() ではその定数名を使って条件分岐する。

    4. 上部の定数だけ変更すれば速度調整できる構造になる。

---

▼ 具体的な方法

    ・加速切替に使う時間条件を「STAGE1_DURATION」「STAGE2_DURATION」として定数化した。

    ・各段階の待機時間を「STAGE1_INTERVAL」「STAGE2_INTERVAL」「STAGE3_INTERVAL」として定数化した。

    ・getInterval() の中で数字を直置きせず、定数名を使って判定する形に修正した。

    ・数値の意味がコード上で見えるようになり、修正箇所も解りやすくなった。

---

▼ 使用したコード・技術

    ・const

    ・定数化

    ・マジックナンバーの排除

    ・条件分岐の整理

    ・保守性向上

---

▼ 重要なポイント

    ・数字を直置きすると、その数値が何を表しているのか後でわかりづらい。

    ・定数名を付けることで「加速時間」「待機時間」の意味がすぐにわかる。

    ・後で速度を調整したいときに修正箇所をすぐに見つけられる。

    ・機能追加時にも条件を増やしやすい。

---

▼ 学んだこと

    ・動くだけのコードではなく、後から読んでも意味が解るコードにするには定数化が重要だと理解した。

    ・数字の直置きを減らすことで保守しやすい構造を作れる

### ② JavaScriptからCSSアニメーションを発火する数値変化処理

▼ 目的

    カウント数値は更新されていたが、数字が切り替わるだけでは変化がわかりづらく、ユーザーが「押した反応」を視覚的に感じにくかった。

    そこで、数値が変わるたびに少し拡大して戻るアニメーションを付け、数値更新を見た目でもわかるようにすることを目的とした。

---

▼ 仕組み

    今回のアニメーションは、JavaScriptが直接数字を動かしているわけではない。
    流れは以下の通り。

    1. CSSで .pop クラスにアニメーション内容を定義する

    2. JavaScriptで count 表示要素に .pop を付ける

    3. CSS側がそのクラスを見てアニメーションを実行する



    つまり、
        ・JavaScript = アニメーション開始の命令
        ・CSS = 実際の見た目の動き
    という役割分担になっている。

---

▼ 具体的な方法

    ・@keyframes で拡大 → 元に戻る動きを作成

    ・ .pop クラスに animation を設定

    ・updateDisplay() の中で animateCount() を呼び出す

    ・animateCount() 内で countEl に pop クラスを付け直す

    長押し連続カウント時でも毎回アニメーションが再生されるように、一度 remove してから add し直す方法を使った。

---

▼ 使用したコード・技術

    CSS

        .pop {
            animation: popAnim 0.2s ease;
        }

    JavaScript

        function animateCount() {
            countEl.classList.remove("pop");
            void countEl.offsetWidth;
            countEl.classList.add("pop");
        }

---

▼ 重要なポイント

    同じクラスを連続で add しても、ブラウザが「変化なし」と判断するとCSSアニメーションが再実行されない場合がある。

    そのため、
        ・一度クラスを外す
        ・ブラウザに外れたことを認識させる
        ・再度クラスを付ける
    という順番が必要になる。

    この考え方により、長押し高速連打中でも安定して数値アニメーションが動作する。

---

▼ 学んだこと

    JavaScriptは見た目そのものを作るのではなく、CSSクラスを操作して見た目の変化を発火させる役割も持てることを学んだ。

    また、連続実行される処理では「命令を書く」だけでなく、ブラウザがどう認識するかまで考える必要があると理解した。
